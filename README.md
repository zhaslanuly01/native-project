# News App (React Native / Expo)

Мобильное приложение для просмотра новостей с авторизацией (биометрия/демо), избранным, локальными уведомлениями, экспортом статьи в JSON, импортом статьи из JSON и скачиванием изображения статьи.

## О проекте

Приложение получает новости из **News API** и предоставляет удобный мобильный интерфейс для:

- просмотра списка новостей,
- фильтрации по категории и дате,
- поиска,
- просмотра деталей статьи,
- открытия оригинала статьи во встроенном `WebView`,
- добавления в избранное,
- импорта/экспорта статей в JSON,
- скачивания изображения статьи,
- получения локальных уведомлений.

## Архитектура проекта

Проект организован в стиле **FSD (Feature-Sliced Design)** с адаптацией под React Native.

### Почему выбран такой подход

FSD помогает:

- разделить ответственность между слоями,
- упростить поддержку и масштабирование,
- сделать код более предсказуемым и переиспользуемым.

### Слои

#### `app`

Глобальная конфигурация приложения:

- `StoreProvider` (Redux store)
- `bootstrap` (инициализация сессии, уведомлений)
- `router` (stack + tabs, auth-flow)

#### `pages`

Экранные компоненты:

- `auth`
- `news-list`
- `news-details`
- `favorites`

#### `widgets`

Составные UI-блоки страниц:

- панель фильтров
- блок действий на деталях статьи
- блок контента статьи
- WebView viewer

#### `features`

Пользовательские сценарии (use cases):

- вход по биометрии
- logout
- переключение избранного
- скачивание изображения
- экспорт JSON
- импорт JSON-файла
- фильтры новостей (UI + типы/константы)

#### `entities`

Бизнес-сущности:

- `article`
- `favorite`
- `session`
- `file` (типы для файловых операций)

#### `shared`

Переиспользуемая инфраструктура:

- `api` (RTK Query baseApi)
- `config` (ENV)
- `lib` (notifications, files, hooks)
- `types` (router types)
- `ui` (Page, Loader, ErrorView)

## Обязательные функции и их реализация

Ниже описаны ключевые обязательные функции и как они реализованы в проекте.

### 1) Отправка файлов с устройства (обязательное)

Реализовано через фичу **`features/upload-file`**.

#### Что делает

- позволяет выбрать файл с устройства (`pickFile`)
- проверяет размер (лимит до 10MB)
- проверяет тип файла (`.json` / mime type)
- читает содержимое файла
- парсит JSON
- валидирует структуру
- импортирует статью в избранное (`favorites`)
- сохраняет обновлённый список в `AsyncStorage`

#### Обработка ошибок

Поддерживаются и маппятся пользовательские ошибки:

- файл слишком большой
- отмена пользователем
- неверный тип файла
- некорректный JSON
- неверная структура JSON
- ошибка чтения файла
- неизвестная ошибка

#### Почему это считается “отправкой файла”

В учебных/демо задачах этот сценарий покрывает пользовательский поток выбора и обработки файла с устройства.
При необходимости легко заменить mock/локальную обработку на реальный HTTP upload к серверу (через `fetch`/`FormData` или отдельный endpoint).

---

### 2) Загрузка (скачивание) файлов с сервера + обработка ошибок (обязательное)

Реализовано через фичу **`features/article-download-image`** + `shared/lib/files`.

#### Что делает

- берёт `urlToImage` из статьи
- скачивает файл по URL (`downloadRemoteFile`)
- сохраняет его на устройство
- сохраняет изображение в галерею (`saveImageToGallery`)
- показывает пользователю результат (успех/ошибка)

#### Обработка ошибок

- отсутствие изображения у статьи
- ошибка скачивания файла
- ошибка сохранения на устройство/в галерею
- отображение ошибок через `Alert` и текст под кнопкой

---

### 3) Хранение данных на устройстве

#### Избранное

- `AsyncStorage`
- хранение списка избранных статей (`favorite_articles_v1`)

#### Сессия / настройки сессии

- `expo-secure-store`
- хранение признака авторизации и настройки биометрии

---

### 4) Авторизация / сессия

Реализовано через:

- `features/biometric-login`
- `entities/session`
- `app/bootstrap/SessionBootstrap`
- `app/router/RootNavigator`

#### Что происходит

- при запуске восстанавливается сессия из `SecureStore`
- пока данные не загружены — показывается `Loader`
- если не авторизован — `AuthPage`
- если авторизован — основной интерфейс (tabs + stack)

---

### 5) Работа с API

Реализовано через:

- `shared/api/baseApi`
- `entities/article/api/articleApi`

#### Почему RTK Query

Выбран RTK Query, потому что он:

- уменьшает количество boilerplate-кода
- удобно управляет loading/error/data
- поддерживает кэширование
- хорошо интегрируется с Redux Toolkit

---

### 6) Уведомления

Реализовано через `shared/lib/notifications` + `NotificationsBootstrap`.

#### Возможности

- запрос разрешений на уведомления
- настройка Android notification channel
- локальные уведомления:

  - при добавлении в избранное
  - при появлении новых новостей

## Установка и настройка

### 1. Клонировать репозиторий

```bash
git clone <URL_репозитория>
cd <название-проекта>
```

### 2. Установить зависимости

```bash
npm install
```

или

```bash
yarn
```

### 3. Создать `.env`

Создайте файл `.env` в корне проекта и добавьте:

```env
EXPO_PUBLIC_NEWS_API_BASE_URL=https://newsapi.org/v2
EXPO_PUBLIC_NEWS_API_KEY=ВАШ_NEWS_API_KEY
```

> Если `EXPO_PUBLIC_NEWS_API_KEY` не задан, приложение покажет предупреждение в консоли.

### 4. Получить API ключ

Получите ключ на News API и подставьте в `.env`.

## Особенности реализации (пояснения по решениям)

### 1. Разделение на `entities / features / widgets / pages`

Это упрощает поддержку:

- бизнес-модели не смешиваются с UI-экраном
- пользовательские сценарии инкапсулированы в `features`
- страницы собирают готовые блоки

### 2. Redux Toolkit + RTK Query

Использован единый подход для:

- локального состояния (`session`, `favorites`)
- серверного состояния (`articleApi`)

Это снижает сложность и делает код более единообразным.

### 3. Безопасное хранение сессии

Данные сессии сохраняются в `SecureStore`, а не в обычном `AsyncStorage`, что лучше подходит для auth-related данных.

### 4. Явная обработка ошибок файлового импорта

В `upload-file` сделан отдельный формат ошибок (`FileTransferError`) и маппинг кодов в понятные сообщения для пользователя.

### 5. Bootstrap-инициализация

Отдельные `SessionBootstrap` и `NotificationsBootstrap` позволяют:

- не перегружать `App.tsx`
- централизованно запускать init-логику
- легко добавлять новые bootstrap-модули
